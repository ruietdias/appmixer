This is a demo of the Appmixer SDK. For more information, visit our online documentation.

Welcome to "YU", the imaginary SaaS product with Appmixer No-Code Workflow components embedded!

Getting started:

  1. Study the source code of this demo (especially the main.js file) to learn how you can embed Appmixer in your own product.
  2. You'll notice the first three lines in the main.js file are the basic configuration lines which you need to provide before going further. The lines look like this: 
                                var BASE_URL = 'YOUR_APPMIXER_API_URL';
                            var USERNAME = 'YOUR_USER_USERNAME';
                            var PASSWORD = 'YOUR_USER_PASSWORD';
    • Replace YOUR_APPMIXER_API_URL in BASE_URL with the Appmixer API URL that you were given during the sign-up process.
    • Replace USERNAME and PASSWORD with some username and password of your test user (e.g. mytest@example.com/mytest123). Note that the demo app (main.js) is implemented so that when this user does not exist, it will create one automatically with the username/password specified. Also note that in your own product, you'll most likely use an email address of your real user as the USERNAME and an auto-generated token as PASSWORD and store these with your real user accounts metadata. That way, you'll be able to associate your own user accounts with the Appmixer "virtual users".
    Save the main.js file and refresh the page.
  3. Click on the "App Studio" menu item to explore some of the embedded Appmixer UI widgets. The widgets used in this demo are:
    • appmixer.ui.FlowManager (App Studio -> My Flows) ... which displays the list of flows the user has previously created. More information about FlowManager can be found in the SDK documentation.
    • appmixer.ui.Designer (App Studio -> My Flows -> Create Flow) ... which displays the no-code drag&drop designer of workflow automations. More information about Designer can be found in the SDK documentation.
    • appmixer.ui.Accounts (App Studio -> Connected Accounts) ... which displays the list of connected accounts (3rd party accounts that the user authenticated to during flow building phase). More information about Accounts can be found in the SDK documentation. Note that this list will be empty until you build a flow that uses a 3rd party app.
    • appmixer.ui.InsightsLogs (App Studio -> Automation Logs) ... which displays the logs representing any activity within the user flows. More information about InsightsLogs can be found in the SDK documentation. Note that the logs will be empty until you start a flow.
  4. Once you're done exploring Appmixer UIs, you might want to try to implement your own custom connectors (either to your private API, any other public 3rd party app or custom utilities). Please follow our online tutorial to learn more.

Note that this demo page references the Appmixer JavaScript SDK from https://my.appmixer.com/appmixer/appmixer.js. However, you might want to replace the my.appmixer.com with the location of your own Appmixer instance (provided to you during the sign-up process) OR download the JavaScript SDK to your own custom location (CDN, Web server). This is recommended to avoid version incompatibilities between your own Appmixer instance engine API and the Appmixer JavaScript SDK.

If at any point during discovering Appmixer you're unsure what to do, please contact our customer support at support@appmixer.com.
